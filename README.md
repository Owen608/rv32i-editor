# RV32I 在线编辑器 Demo

本地可运行的 RV32I 汇编编辑 + 模拟器 Demo，支持内置汇编器（无需安装工具链）或使用 RISC-V 工具链，并提供**内置简易 C 编译器**将 C 子集编译为 RV32I 汇编。

**当前状态**：调试与 RV32I 初步测试已完成；接下来需进行详细测试与边界用例验证。

---

## 一、支持的模式

### 1. C → 汇编（内置简易 C 编译器）

- 在「C 源代码」框中编写 C 子集代码，点击「C → 汇编」后，页面内将 C 编译为 RV32I 汇编并填入「汇编源码」框，再经内置汇编器生成机器码。
- **C 语法支持子集**：`int main()`、标量/数组声明与赋值、`for`（`i < N` / `i <= N`）、`if`（单一比较或 `&&` / `||`）、`return`；数组形式如 `a[i] = a[i-1] + i`、`a[i] = a[i-1] * 常数 + i`、`sum += a[i]` 等。多 `main()` 时编译**最后一个** `main()`。
- 详见下文「C 子集与编辑器功能」。

### 2. 汇编 / 编译模式

| 模式 | 说明 | 依赖 |
|------|------|------|
| **内置汇编器** | 使用页面内 JS 汇编器，将汇编源码转为机器码 | 无，打开页面即可用 |
| **工具链** | 调用本机 `riscv-none-elf-as` 汇编，支持完整 GAS 伪指令与语法 | 需安装 RISC-V GNU 工具链并加入 PATH |

- 在页面顶部选择「内置汇编器」或「工具链」后，点击「汇编 / 编译」即可。
- 汇编成功后，右侧显示**机器码**（十六进制）与**反汇编**。

### 3. 执行模式

| 模式 | 说明 |
|------|------|
| **单步执行** | 每点击一次「单步」执行一条指令，并刷新寄存器与 PC |
| **连续执行** | 点击「连续」后自动反复单步，直到遇到 ecall exit 或点击「停止」 |
| **重置** | 点击「重置」后重新加载当前机器码，PC 与寄存器恢复初始状态 |

- 执行前需先「汇编」再「加载到模拟器」；未加载时点「单步」会提示先加载。

### 4. 其他功能

- **寄存器查看**：执行区下方显示 32 个通用寄存器（x0~x31，含 ABI 名）及当前值（十六进制）。
- **PC 查看**：显示当前程序计数器（如 `0x80000000`）。
- **机器码 / 反汇编**：每条指令一行，格式为 `地址  机器码(32位)  反汇编`。

## 二、支持的指令

### 1. 内置汇编器支持的指令

以下为**内置汇编器**（选择「内置汇编器」时）可识别的助记符；未列出的助记符会报「不支持的指令或伪指令」。

#### 1.1 基指令（RV32I）

**算术 / 逻辑 — 立即数（I-type, opcode 0x13）**

| 助记符 | 格式 | 说明 |
|--------|------|------|
| addi | rd, rs1, imm | imm 为 12 位有符号 (-2048~2047) |
| andi | rd, rs1, imm | |
| ori | rd, rs1, imm | |
| xori | rd, rs1, imm | |
| slli | rd, rs1, shamt | shamt 0~31 |
| srli | rd, rs1, shamt | |
| srai | rd, rs1, shamt | |

**算术 / 逻辑 — 寄存器（R-type, opcode 0x33）**

| 助记符 | 格式 | 说明 |
|--------|------|------|
| add | rd, rs1, rs2 | |
| sub | rd, rs1, rs2 | |
| and | rd, rs1, rs2 | |
| or | rd, rs1, rs2 | |
| xor | rd, rs1, rs2 | |
| sll | rd, rs1, rs2 | |
| srl | rd, rs1, rs2 | |
| sra | rd, rs1, rs2 | |

**访存 — 取数（I-type, opcode 0x03）**

| 助记符 | 格式 | 说明 |
|--------|------|------|
| lb | rd, imm(rs1) | 字节有符号 |
| lh | rd, imm(rs1) | 半字有符号 |
| lw | rd, imm(rs1) | 字 |
| lbu | rd, imm(rs1) | 字节无符号 |
| lhu | rd, imm(rs1) | 半字无符号 |

**访存 — 存数（S-type, opcode 0x23）**

| 助记符 | 格式 |
|--------|------|
| sb | rs2, imm(rs1) |
| sh | rs2, imm(rs1) |
| sw | rs2, imm(rs1) |

**分支（B-type, opcode 0x63）**

| 助记符 | 格式 | 说明 |
|--------|------|------|
| beq | rs1, rs2, offset | offset 可为标签或数字 |
| bne | rs1, rs2, offset | |
| blt | rs1, rs2, offset | 有符号 < |
| bge | rs1, rs2, offset | 有符号 ≥ |

**跳转**

| 助记符 | 格式 | opcode |
|--------|------|--------|
| jal | rd, offset | 0x6f (J-type) |
| jalr | rd, rs1, imm | 0x67 (I-type) |

**高位立即数（U-type）**

| 助记符 | 格式 | opcode |
|--------|------|--------|
| lui | rd, imm | 0x37 |
| auipc | rd, imm | 0x17 |

**系统 / 环境**

| 助记符 | 格式 | opcode |
|--------|------|--------|
| ecall | | 0x73, imm=0 |
| ebreak | | 0x73, imm=1 |

#### 1.2 伪指令（内置汇编器展开）

| 伪指令 | 等价基指令 |
|--------|------------|
| nop | addi x0, x0, 0 |
| mv rd, rs | addi rd, rs, 0 |
| li rd, imm | addi rd, x0, imm（\|imm\|≤2047）；否则 lui + addi |
| la rd, imm/label | 载入地址：小立即数用 addi；否则 lui + addi（支持如 `0x80000000`） |
| ret | jalr x0, ra, 0 |

#### 1.3 指示性语句（仅忽略，不生成代码）

- `.text`、`.data`、`.globl`、`.globl _start` 等以 `.` 开头的行会被跳过，不参与汇编。
- 标签（如 `_start:`）会参与地址计算，可用于 `beq`/`bne`/`jal` 等的目标。

#### 1.4 内置汇编器不支持的指令

- **分支**：bltu、bgeu（模拟器可执行，但汇编器暂不识别）。
- **比较立即数**：slti、sltiu（汇编器、模拟器均未实现）。
- **部分 M 扩展助记符**：mulh/mulhsu/mulhu/divu/remu（模拟器可执行，但内置汇编器暂不识别）。

### 2. 模拟器支持的指令（执行阶段）

模拟器按**机器码**执行，因此只要机器码正确（例如由「工具链」生成），即使内置汇编器不识别某助记符，模拟器仍可执行。下面列出**按 opcode 实际会执行的指令**。

#### 2.1 RV32I

- **LUI / AUIPC** (0x37, 0x17)
- **JAL / JALR** (0x6f, 0x67)
- **分支**：beq, bne, blt, bge, bltu, bgeu (0x63, funct3 0~7)
- **取数**：lb, lh, lw, lbu, lhu (0x03, funct3 0,1,2,4,5)
- **存数**：sb, sh, sw (0x23, funct3 0,1,2)
- **I-type 算术/逻辑**：addi, slti, sltiu, xori, ori, andi, slli, srli, srai (0x13, funct3 0~7)
- **R-type 算术/逻辑**：add, sub, sll, slt, sltu, xor, srl, sra, or, and (0x33, 对应 funct3/funct7)
- **ECALL / EBREAK** (0x73)

#### 2.2 系统调用（ecall）

| a7 值 | 含义 | 行为 |
|-------|------|------|
| 64 | — | 本实现中 a0 置 0 |
| 93 | exit | 程序结束，停止执行 |

其他 a7 值不处理，仅 pc+4 继续执行。

#### 2.3 M 扩展（整数乘除法，模拟器支持）

- mul, mulh, mulhsu, mulhu  
- div, divu  
- rem, remu  

这些指令若由「工具链」生成（`-march=rv32im`），模拟器会按 M 扩展语义执行；**内置汇编器不识别** M 扩展助记符。

## 三、环境与运行

### 环境要求

- **Node.js**：v20+，无需额外 npm 包（`server.js` 仅用内置模块）。
- **RISC-V 工具链**（可选）：仅在使用「工具链」模式时需要，如 xPack `riscv-none-elf-gcc`（含 as），将 `bin` 加入 PATH。

### 快速运行

**方式一：直接打开（仅内置汇编器）**

1. 用浏览器打开项目根目录下的 `index.html`（file://）。
2. 选择「内置汇编器」，编写汇编 → 「汇编 / 编译」→ 「加载到模拟器」→ 「单步」或「连续」。

**方式二：启动本地服务（推荐）**

```bash
cd g:\riscv-project
node server.js
```

浏览器访问 **http://localhost:3000**。使用「工具链」时需本机已安装并配置好 RISC-V 工具链。

### Windows 下安装 RISC-V 工具链（可选）

1. 打开 [xPack riscv-none-elf-gcc releases](https://github.com/xpack-dev-tools/riscv-none-elf-gcc-xpack/releases)。
2. 下载 Windows 版 zip，解压到目录（如 `C:\riscv-toolchain`）。
3. 将 `解压目录\bin` 加入系统 PATH。
4. 命令行执行 `riscv-none-elf-as -v` 能输出版本即安装成功。

## 四、C 子集与编辑器功能

### 1. C 语法子集（内置编译器）

- **程序**：仅处理最后一个 `int main() { ... }`，花括号必须成对。
- **类型与变量**：`int` 标量、`int a[N]` 数组；变量分配在 a0–a7，数组基址 0x80000000。
- **控制**：`for (int i = 初值; i < N 或 i <= N; i++) { ... }`、`if (条件) { ... }`，条件可为单一比较（`==` / `!=` / `<` / `>` / `<=` / `>=`）或 `( cond && cond )` / `( cond || cond )`。
- **运算与赋值**：`+=`、`x++`/`x--`、`x = a[i]`、`a[i] = 值`、`a[i] = a[i-1] + x`、`a[i] = a[i-1] * 常数 + x`、`sum += a[i]`、二元运算与 `return 变量`。

### 2. 编辑器功能

| 功能 | 说明 |
|------|------|
| **自动补全** | C 源码可选「自动补全」：输入 `(` `[` `{` 时自动补全 `)` `]` `}`，输入右括号时若右侧已是同种括号则跳过。 |
| **Tab 空格** | C 与汇编编辑区按 Tab 插入空格（不跳焦点）；可选 **Tab 宽度**：2 格 / 4 格 / 8 格。 |
| **Enter 在 `{}` 内** | C 源码中，光标在同行 `{` 与 `}` 之间按 Enter：自动插入换行 + 多一级缩进，再换行 + 与行首对齐的 `}`，光标停在中间空行。 |

- C 源码区有「自动补全」勾选与「Tab 宽度」下拉；汇编区有「Tab 宽度」下拉。

## 七、I/O（stdin/stdout）与 ecall 扩展

执行区新增了 **输入（stdin）** 与 **输出（stdout）** 窗口，用于测试 ecall I/O。

### 1) ecall（a7）约定

| a7 值 | 类型 | 含义 | 约定/行为 |
|------:|------|------|-----------|
| 1 | 读 | readInt | 从输入整数队列取一个 32 位整数，写入 a0；无输入则暂停等待 |
| 2 | 输出 | printInt | 输出 a0（有符号 32 位）到 stdout，并追加换行 |
| 3 | 读 | readChar | 从输入字节队列取一个字节，写入 a0（0~255）；无输入则暂停等待 |
| 4 | 输出 | printChar | 输出 (a0 & 0xFF) 对应字符到 stdout |
| 5 | 读 | readString | 参数：a0=addr（数据区地址）、a1=maxLen；从输入字节队列写入内存，读到 `\\0` 结束并写入 `\\0`；返回写入长度（不含 `\\0`）到 a0；无输入则暂停等待 |
| 6 | 输出 | printString | 参数：a0=addr（数据区地址）；从内存读取字节直到 `\\0`，输出到 stdout |
| 93 | — | exit | 停止执行 |

> 注意：`readString/printString` 的地址建议使用数据区（默认从 `0x80000000` 起），否则写入会被忽略或读到 0。

### 2) 前端输入框用法

- **发送整数**：把输入框内容按空格/换行解析成整数队列（供 `a7=1` 使用）
- **发送文本**：把输入框内容按 ASCII 转成字节队列（供 `a7=3/5` 使用），可勾选「末尾追加 \\0」

### 3) 示例

`example_ecall_io.s` 演示了 readInt/printInt、readChar/printChar、readString/printString（缓冲区使用 `0x80000000`）。
